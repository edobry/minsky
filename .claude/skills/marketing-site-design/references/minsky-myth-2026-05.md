# Minsky myth workshop — 2026-05-19

Worked example for the `marketing-site-design` skill's Section 11 process. End-to-end walk through Steps 1-5 of the myth-selection workshop, with Minsky as the brand. Run on 2026-05-18/19 between Eugene (principal) and a Claude Opus 4.7 session in mt#1927.

Use as the worked reference when running this process for any future Minsky marketing surface or for an adjacent brand.

---

## Step 1 — Category-default myth

**Question:** What proposition do AI-product buyers already accept as obvious before they encounter Minsky?

**Locked statement:**

> _"AI agents are productivity tools you wire to your existing workflow. The job is choosing the right agent and giving it the right tools."_

This default naturalizes:

- Agents as **enhancements to the individual engineer** (Cursor's frame)
- Agents as **plumbing between an agent and SaaS apps** (Composio's frame)
- Agents as **factory workers operated by a foreman** (Factory's frame)
- The unit of work as a _task an individual engineer hands off_
- The buyer's question as _"which agent product helps me / my team get more done?"_

A buyer landing on Minsky's site has this default loaded. Whatever Minsky says will be heard through this filter unless the site explicitly displaces the default.

## Step 3 — Test each candidate; lock one

Tests applied:

| Test                              | Question                                                                                              |
| --------------------------------- | ----------------------------------------------------------------------------------------------------- |
| Contestable                       | Would a buyer disagree before encountering the site? Would a competitor find it awkward to claim?     |
| Carried by product                | Can the product surfaces (CLI / cockpit / reviewer-bot / mesh / asks / memory) instantiate this myth? |
| Durable                           | Will it still be true and important in 2-3 years?                                                     |
| Aligned with principal investment | Does the operator want to spend years naturalizing this proposition?                                  |

### Results

- **#1 (Exocortex / flock)** — strongest contestability (the category default is "tools," not "extension"), strongest aesthetic resonance with Eugene's existing corpus, carriable by the product, durable. **PICKED.** Risk: "exocortex" is unfamiliar vocabulary for fresh audiences; needs careful instantiation via residual codes (Section 4 of SKILL.md). Risk mitigated via Step 4 below.
- **#2 (Cybernetic body)** — academically clean, maps to VSM, durable. Misses the _enchantment_ the corpus is reaching for. Available as secondary framing.
- **#3 (Operative ontology)** — most ambitious; explicitly invokes magick. Risk: alienates the engineer-buyer who reads "magick" as woo. **Footnote-only**, not foreground.
- **#4 (Locus)** — accurate to the corpus but the principal's existing memory `Locus and Minsky long-term convergence` says Locus is not a present-day pitch frame. **Deferred** per existing discipline.
- **#5 (Servitor)** — most precise per the Operative Ontology vocabulary. Risk: "servitor" reads as occult to engineers. **Manifesto-page only**, not foreground.

### Locked myth (single declarative sentence)

> **_"Minsky is the exocortex for software organizations led by a principal — a flock of intent-bearing servitors that extends the principal's cognition into coordinated realized work."_**

Underlying proposition: AI agents are autonomous intent-bearers that extend the principal's cognition. Minsky is the substrate that makes the flock coherent — the cybernetic body that translates declared intent into realized work.

### Code name

**Cyberbrain / Section 9** — autonomous-flock cybernetic substrate that extends a principal's cognition, rendered as a serious operational profession.

## Step 5 — Visual specification

Derived in `SKILL.md` Section 8 (the Minsky-specific layer). Summary of the locks:

- **Idiom:** B (product-screenshot dominant, restrained)
- **Stack:** Astro (or equivalent static-output framework). The marketing site rebuild deferred to a downstream task.
- **Typography:** Geist (free, 400 weight, tight tracking) + JetBrains Mono for eyebrows + warmer mono (Berkeley Mono / IBM Plex Mono italic) for "system speaks" surfaces
- **Color:** near-black background + near-white text + cyan signal accent + sparing amber/NERV-red warning + Iso-pastel warmth in companion-personality surfaces
- **Layout:** left-caption + right-product-screenshot, alternating, no centered hero, no sticky-numbered-nav-with-colored-panels

- **Product surfaces:** 8 specific scenes from real Minsky (CLI / MCP call / cockpit / reviewer-bot / task graph / memory recall / hook denial / asks)
- **Voice:** Macx prose register; Magilumiere tonal lock; 3-4 word conceptual section headlines; sentence case; peer-to-peer
- **Co-product logos** in place of customer logos: Claude Code / Cursor / Codex / MCP / GitHub / Notion / Railway

## pee_zombie corpus discovery

During the workshop (2026-05-19), Eugene mentioned a five-year Twitter corpus (8,094 original tweets, 2020-08 → 2025-04, @pee_zombie) as background substrate informing his current thinking. The workshop session extracted the archive's `data/tweets.js` into an ephemeral temp-folder JSON and grep-scanned for theme alignment; mt#1930 subsequently shipped the durable query surface (see Cross-references below).

Three load-bearing tweets surfaced that show the brand thesis was already operating substrate in the corpus years before the workshop:

1. **2022-06-24, exocortex-as-tribe (flock as social substrate):**

   > _"a tribe is an exocortex, an extension of our Self into the social connectome; this is socially-embodied [cognition]"_

2. **2023-05-08, cognitive engineering = applied cybernetics:**

   > _"cognitive engineering is a modern revival of applied cybernetics, which thru a systems frame studied the information processing dynamics of various realworld systems — less a metaphor & more so a literal truth: we are engaged in nothing [less than that]"_

3. **2022-04-01, Operative Ontology corpus as a tweet:**
   > _"magick is any tool you can use without understanding its operating principles. magick operates on interface (ritual) alone. magick is a reified abstraction, a stable world-compression"_

**Implication for the brand:** the cyberbrain / Section 9 / exocortex frame is **not being grafted onto Minsky from the outside**. It has been the operating system of the principal's thinking for half a decade. The marketing site is recovering a coherence that already exists in the corpus.

## The "drawn from, not literally" discipline as applied to Minsky

Per Section 4 of SKILL.md (bridge-as-affect), specific rules for how the five-layer references show up on the Minsky site:

### What goes on the site

- Typography that pattern-matches to GitS Section 9 ops register (mono eyebrows + light grotesque)
- Color palette that pattern-matches to Section 9 + Eva (near-black + cyan signal + amber warning)
- Magi-aesthetic warmth in "system speaks" surfaces (Eva-coded but not depicting Eva)
- HUD-style status overlays in product screenshots (cyberbrain-link aesthetic, not labeled as such)
- Sync-rate gauges as a recurring micro-instrument (Eva-coded without depicting the cockpit)
- Iso-pastel softness in agent-companion surfaces (Iso-coded without depicting characters)
- Macx-register prose in manifesto / About copy (literary register without quoting Accelerando)
- Magilumiere-style "this is a serious profession" tonal seriousness (without any magical-girl imagery)
- A literary epigraph somewhere on the site, possibly from Macx, naming the flock proposition
- Stand Alone Complex as a named operating concept the brand can claim (carefully, not as flag-waving reference)

### What never appears on the site

- Literal Tachikomas, Major, Section 9 characters, or GitS still frames
- Eva units, Shinji/Asuka/Rei, NERV terminal imagery
- Magical girls, Magilumière protagonists, sigils-as-pastiche
- Iron Man / JARVIS visual cues (rejected — overdone)
- Pacific Rim Jaeger / Drift visuals (rejected — principal unfamiliarity)
- Mecha as a visual element
- Anime stills, anime fanart, anime stylization
- Skynet/Terminator-coded futurism (wrong cultural code entirely)

### Heuristic at point-of-decision

Would the right reader recognize the borrowing without being told? If yes, ship it. If you have to caption the reference for it to read, it's gone literal — cut and replace.
